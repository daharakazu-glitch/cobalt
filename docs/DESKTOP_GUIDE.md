# Mac デスクトップ整理ガイド（AIチーム作業環境）

ユーザーのMac上のデスクトップを、**どのAIモデル（Fable / Opus / Sonnet 等）の
セッションでも迷わず作業を継続できる**ように整理するための手順書。

> このファイルはリポジトリ（`daharakazu-glitch/cobalt`）の `docs/` にあり、
> クラウドセッションからも読める。Mac側の整理はユーザー自身か、
> **Mac上で起動した Claude Code セッション**が実施する（クラウドからは不可）。

---

## 1. 動かしてはいけないもの（★最重要）

| パス | 理由 |
|---|---|
| `/Users/kazuyukiharada/Desktop/AIチーム/` | **フォルダ名・場所を変えない**。Claude の自動メモリ（`~/.claude/projects/…/memory/MEMORY.md`）がこのパスに紐づいており、改名・移動すると過去の記憶が自動ロードされなくなる。 |
| `/Users/kazuyukiharada/Desktop/AIチーム/grammar-app/` | git リポジトリ本体（GitHub の `cobalt`）。場所を変えると git は動くが、Claude のプロジェクト履歴・メモリとの紐づけが切れる。**そのまま**。 |
| `grammar-app/` の中身 | 公開サイトそのもの。中の整理はリポジトリ側の作業（ブランチ＋PR）で行う。Finder で直接動かさない。 |

**方針: 「AIチーム」と「grammar-app」は現状の場所に固定し、その“周り”を片付ける。**

## 2. 目標の形

### `Desktop/AIチーム/` の中

```
Desktop/AIチーム/
├── grammar-app/   ← ★そのまま（gitリポジトリ。触らない）
├── 00_inbox/      ← とりあえず置く場所（未分類の一時置き場）
├── 10_docs/       ← メモ・下書き・企画資料（※仕様の正本はリポジトリの docs/HANDOVER.md）
├── 20_assets/     ← 画像・スクショ・素材の元データ
└── 90_archive/    ← 終わったもの・旧版（消す前の待避場所）
```

- 番号プレフィックス（00/10/20/90）は Finder で常に同じ順に並ばせるため。
- **仕様・進捗の「正本」はあくまでリポジトリの `docs/HANDOVER.md`**。
  `10_docs/` はあくまで下書き置き場で、確定した内容はリポジトリへ反映する。

### デスクトップ直下（AIチーム以外に散らかっている場合）

```
Desktop/
├── AIチーム/      ← ★そのまま
├── _整理箱/       ← 散らかったファイルを一旦全部ここへ
└── （常用するものだけ残す）
```

一旦 `_整理箱/` に集めてから、後日中身を `AIチーム/` 配下の適切な場所か
書類フォルダへ振り分ける（いきなり削除しない）。

## 3. 手作業でやる場合のコマンド

Mac のターミナルで（**mv のみ・削除はしない**ので安全）:

```bash
cd ~/Desktop/AIチーム
mkdir -p 00_inbox 10_docs 20_assets 90_archive

# 中身を確認してから、grammar-app と新規フォルダ以外を振り分ける
ls -la
# 例: mv 企画メモ.txt 10_docs/
# 例: mv スクショ*.png 20_assets/
# 迷ったら: mv <ファイル> 00_inbox/
```

デスクトップ直下の散らかりは:

```bash
mkdir -p ~/Desktop/_整理箱
# 例: mv ~/Desktop/*.png ~/Desktop/_整理箱/
```

## 4. Mac の Claude Code に任せる場合（推奨）

Mac で `~/Desktop/AIチーム` を開いた Claude Code セッションに、
以下をそのまま貼り付ければよい（モデルは Opus 等どれでも可）:

> デスクトップの整理をお願いします。手順書はリポジトリ（grammar-app）内の
> `docs/DESKTOP_GUIDE.md` にあります。それを読んで、§1 の「動かしてはいけないもの」
> を厳守しつつ、§2 の目標の形に `~/Desktop/AIチーム` と Desktop 直下を整理して
> ください。ファイルの削除はせず、移動のみ。移動先に迷うものは `00_inbox/` へ。
> 終わったら「何をどこへ動かしたか」の一覧を報告してください。
